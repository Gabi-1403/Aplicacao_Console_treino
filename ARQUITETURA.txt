Questão conceitual, escreva em detalhes ou diagramas como você montaria uma arquitetura para o cenário abaixo, informando as tecnologias, arquitetura, e o que mais achar necessário.
Em tempos de expansão digital, sua empresa foi contratada para desenhar uma arquitetura moderna que sustente o crescimento digital e de vários novos canais, e também tenha formas de manter o legado funcionando. Descreva ou desenhe o que e como você utilizaria para suportar este crescimento, tendo em vista que é necessário uma arquitetura que agregue os meios de comunicação com Mainframe, e que todos os sistemas web possam se comunicar entre eles sem a reescrita de códigos. Todos os sistemas são extremamente críticos e de alta performance, também contando com um volume consideravelmente alto de dados sendo transacionados a todo tempo. Comente também como você desenharia a solução para implantação deste cenário, visando que quanto menor a dependência com áreas de operação 
para executar a implantação na mão, mais produtivo e assertivo será a empresa.


Bando de Dados Oracle: 
Para salvar as informações na base de dados, pois a arquitetura é mais flexível e com mais recursos para otimização de performance. Possui índices que permite otimizar consultas em BD, além de possuir mais recursos de segurança e performance.

Análise de Requisitos:
Para ter uma perspectiva que direciona um andamento de um projeto, dessa forma será possível a identificação de um problema e encontrar possíveis resoluções.

Aps.Net Core C#: 
Uma linguagem popularmente conhecida no mercado que tem uma arquitetura que é voltada para testes, possui uma alta performance e facilita no desenvolvimento para páginas web com interatividades.

ADO.NET Entity Framework:
Para utilização de Links para consultas, armazenar dados em objetos de domínio em um banco de dados de forma automatizada.

Testes Unitários:
Os testes devem ser realizados para que qualidade do sistema seja mantida sem comprometer com outros desenvolvimentos incluídos.

TortoiseSVN:
Um sistema de Controle de Versão, onde realizado o merge dos códigos, pode ser feito comparações entre código desenvolvidos antes e o depois, facilitando consultas rápidas. 

Plataforma de Pacotes:
Alguma plataforma pública ou até mesmo desenvolvida pela própria empresa, para importar os pacotes gerados, para que esses pacotes gerados sejam registrados com data e número de versão. Assim o próprio desenvolvedor ou até mesmos o cliente poderá fazer o download desse pacote para atualizar o ambiente.
